# archive layout

- Use `runbooks/<os>/...` for reusable validated procedures.
- Use `plans/` for unverified or future work.
- Use `executions/YYYY/MM/YYYY-MM-DDTHHmm+ZZZZ-short-slug/` for real executions.
- Keep `README.md` as the execution entrypoint.
- Keep reviewed evidence in `evidence/` and raw logs or generated files in `artifacts/`.
- Run archive validation before final delivery.
