# runtime-repair

## Use Cases

Agent CLI launch failures, broken package-manager shims, hook stdin encoding bugs, auto-update state review, and version triage.

## Non-Use Cases

Repo business logic changes, broad tool upgrades without evidence, or editing user runtime configuration before the failing layer is known.

## Supported OS

Windows, macOS, and Linux. Windows checks should treat PowerShell, Git Bash, package shims, and child-process encoding as separate evidence.

## Inputs

Exact command, shell, error text, runtime path, hook payload if relevant, package manager, and authorization boundary.

## Outputs

Root-cause layer, readonly evidence, recommended repair, skipped mutations, and verification command.

## Execution Steps

Run runtime-check, inspect launch path and package tree, reproduce encoding-sensitive failures with generated bytes, then propose a targeted repair.

## Human Confirmation Points

Package reinstall, config edits, auto-update changes, PATH changes, hook rewrites, and global environment mutations.

## Failure Handling

If evidence differs by shell, keep both results and do not infer a single cause until the actual executable and child-process environment are known.

## Example Prompts

- "Diagnose why the agent CLI stopped launching after reboot; do not reinstall yet."
- "Check whether this hook parse error is encoding or JSON shape."

## Recommended Workflows

runtime-check, preflight
