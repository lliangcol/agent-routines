# desktop-design-system

## Use Cases

Design tokens, theme palettes, dense admin UI, install matrices, task centers, forms, status badges, and desktop productivity visual QA.

## Non-Use Cases

Marketing landing pages, generic AI hero pages, decorative illustration systems, or mobile-first consumer UI.

## Supported OS

Windows, macOS, and Linux. Prefer system fonts and native platform conventions.

## Inputs

Target workflow, platform constraints, theme requirements, language requirements, component inventory, and screenshot evidence.

## Outputs

Design token guidance, component rules, visual QA findings, and concrete layout fixes.

## Execution Steps

Review the workflow, define or check tokens, inspect tables/forms/tasks, verify themes and language lengths, and report visual issues by severity.

## Human Confirmation Points

New component libraries, external design assets, brand changes, and full visual direction changes.

## Failure Handling

If screenshots are unavailable, mark visual QA as partial and require browser or app screenshots before final approval.

## Example Prompts

- "Use `desktop-design-system` to make this Electron UI look like a modern desktop productivity tool."
- "Use `desktop-design-system` to review dark theme tokens and status colors."

## Recommended Workflows

doc-check, gate-check
