# release-guard Checklist

- Identify the release ecosystem and source of truth.
- Check README, license, changelog, security notes, and support docs.
- Inspect package metadata and included files.
- Prefer dry-run commands over real publication.
- Run security-check before public release claims.
- Keep local validation separate from registry or GitHub proof.
- Do not publish, tag, push, or mutate versions without explicit confirmation.
- Report release blockers separately from warnings.
