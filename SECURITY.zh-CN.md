# 安全策略

## 支持版本

Agent Routines 作为一个完整 routine 库统一版本化。公开使用者应从经过审阅的 `vX.Y.Z` tag 安装，而不是直接安装 `main`。安全修复默认进入下一个 tag 版本；如果私下交接更安全，维护者可选择先私下处理。

## 报告漏洞

不要在公开 issue 中提交密钥、针对第三方系统的利用步骤、或私有仓库路径。请通过仓库所有者资料页联系维护者，或在 GitHub private security advisory 可用时使用该渠道。

请提供：

- 受影响的 Skill、workflow、adapter 或文档路径。
- 审查的具体版本、tag 或 commit。
- 影响范围和所需权限。
- 不暴露真实密钥或生产数据的最小复现步骤。

## 敏感信息处理

共享证据前请先脱敏凭据、token、内部主机名、客户数据和私有文件路径。如果真实密钥已经被提交或安装，请先在仓库外轮换密钥，再报告受影响路径和 commit 范围。

## 范围

范围内：仓库脚本、安装适配器、manifest 分发、workflow 安全边界和公开发布产物。

范围外：agent runtime、GitHub、包管理器、操作系统，或用户在这些 routines 之外调用的第三方工具漏洞。

## 响应预期

本项目不承诺固定 SLA。维护者会根据可利用性、暴露面和发布影响进行分级；在安全披露条件允许时，将修复记录到 `CHANGELOG.md`。
